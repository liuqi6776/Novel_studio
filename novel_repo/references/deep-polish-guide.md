# 深度润色指南 - 去除 AI 味

## ⚠️ AI 写作常见问题

AI 生成的内容往往带有明显的"AI 味"，需要重点检查和修改。

## ✅ 润色输出格式

润色结果必须包含三项：

- 问题点
- 证据段落
- 修改建议

---

## 1. 去除过度修饰的形容词

### ❌ AI 常用词堆砌（删除/替换）

| 类别 | AI 常用词 | 建议 |
|------|----------|------|
| 视觉 | 璀璨、瑰丽、绚烂、耀眼、华丽 | 用具体颜色/形状替代 |
| 感觉 | 震撼、惊心动魄、难以言喻 | 用身体反应替代 |
| 情绪 | 复杂的情感、五味杂陈 | 用具体动作替代 |
| 氛围 | 神秘、诡异、阴森 | 用具体细节替代 |

### 示例对比

```
❌ AI 版：
璀璨的阳光洒在瑰丽的宫殿上，绚烂的光芒令人震撼。
她心中涌起复杂的情感，五味杂陈。

✅ 人写版：
阳光照在宫殿的金顶上，反光刺得她眯起眼。
她攥紧了拳头，指甲掐进掌心。
```

### 修改原则

1. **一个场景最多用 1 个形容词**
2. **优先用名词和动词**
3. **用具体事物替代抽象修饰**

---

## 2. 减少抽象陈述

### ❌ 抽象 → ✅ 具体

| 抽象陈述 | 具体动作/对话 |
|----------|--------------|
| "心中涌起复杂的情感" | "她咬住下唇，尝到血腥味" |
| "感到非常紧张" | "手心出汗，在裤子上蹭了蹭" |
| "他很生气" | "他把杯子摔在地上，碎片溅到脚踝" |
| "她犹豫不决" | "手指在门把手上悬了三秒，又放下" |
| "心中一凛" | "后背的汗毛竖了起来" |

### 修改技巧

**问自己：**
- 这个情绪**身体上**什么感觉？
- 这个人会**做什么动作**？
- 会**说什么话**？
- 脸上**什么表情**？

---

## 3. 打破四字格律

### ❌ AI 爱用的四字成语（避免堆砌）

```
心潮澎湃、热血沸腾、惊心动魄
毛骨悚然、不寒而栗、若有所思
意味深长、若有所思、若有所思
```

### 问题

- 陈词滥调，读者无感
- 过于书面化，不自然
- AI 痕迹明显

### 修改示例

```
❌ AI 版：
他心潮澎湃，热血沸腾，心中惊心动魄。

✅ 人写版：
他心跳得厉害，手心全是汗。
"该死。"他骂了一句。
```

### 原则

- **四字成语每章不超过 3 个**
- **优先用口语表达**
- **用动作替代成语**

---

## 4. 增加口语化表达

### ❌ 书面语套话 → ✅ 个性化对话

```
❌ AI 版：
"此事事关重大，容我再三思。"
"阁下所言极是，在下明白。"

✅ 人写版：
"这事儿……我得想想。"
"你说得对，我懂了。"
```

### 对话个性化技巧

**为每个角色定义：**

| 要素 | 示例 |
|------|------|
| **口头禅** | "该死"、"你懂的"、"怎么说呢" |
| **句长** | 短句/长句/断断续续 |
| **语气词** | "吧"、"嘛"、"咯"、"呗" |
| **称呼习惯** | 全名/昵称/外号/不说名字 |
| **说话节奏** | 急促/缓慢/爱打断别人 |

### 示例

```
角色 A（粗鲁直爽）：
"屁话！干就完了，磨叽啥？"

角色 B（谨慎多疑）：
"这个……不太好吧？我是说，万一……你懂我意思吧？"

角色 C（高傲冷漠）：
"呵。随你。"
```

---

## 5. 优化节奏感

### ❌ 句式单调

```
❌ AI 版（全是长句）：
他缓缓地走向那扇古老的大门，心中充满了复杂的情感。
阳光从窗户洒进来，照亮了地面上的灰尘。
她站在那里，看着他的背影，心中涌起一阵悲伤。

✅ 人写版（长短交替）：
他走向大门。
脚步很慢，像拖着什么东西。
阳光从窗户斜进来，灰尘在光柱里打转。
她站着，看他的背影。
心里一空。
```

### 节奏技巧

1. **长句（20 字+）**：描写场景、铺垫氛围
2. **中句（10-20 字）**：叙述动作、推进剧情
3. **短句（10 字内）**：强调、转折、冲击

### 比例建议

```
长句：30%
中句：50%
短句：20%
```

### 检查方法

**朗读出来**——哪里喘不过气，哪里就该断句。

---

## 6. 细节具象化

### ❌ 笼统描述 → ✅ 五感细节

| 感官 | 笼统 | 具象 |
|------|------|------|
| **视觉** | "很漂亮" | "裙摆上绣着银色藤蔓，灯光下泛着冷光" |
| **听觉** | "很吵" | "玻璃碎裂声混着尖叫声，刺耳" |
| **嗅觉** | "很难闻" | "腐烂的木头味混着霉味，呛鼻子" |
| **触觉** | "很冷" | "寒气顺着袖口往里钻，胳膊起了一层鸡皮疙瘩" |
| **味觉** | "很难吃" | "苦得像胆汁，舌尖发麻" |

### 示例对比

```
❌ AI 版：
房间里很乱，有一股难闻的味道。

✅ 人写版：
外卖盒堆在桌上，汤汁凝固成白色油块。
烟灰缸满了，烟头泡在剩咖啡里。
空气里有股酸臭味，像什么东西坏了。
```

### 细节密度

- **关键场景**：3-5 个感官细节
- **过渡场景**：1-2 个细节
- **日常场景**：可省略

---

## 7. 展示而非讲述 (Show, Don't Tell)

### ❌ 直接讲述 → ✅ 动作/对话展示

```
❌ 讲述：
他很紧张。

✅ 展示：
他不停地按圆珠笔，咔哒、咔哒、咔哒。
"你能不能别按了？"同事瞪了他一眼。
他停下，手又开始抖。
```

### 常见"讲述"及"展示"替代

| 讲述 | 展示 |
|------|------|
| "他很累" | "他揉揉眼睛，咖啡已经凉了第三杯" |
| "她很害怕" | "她屏住呼吸，手心里全是汗" |
| "他们关系不好" | "他进来时，她转身看向窗外" |
| "他很富有" | "他随手把车钥匙扔给泊车员，没看是什么车" |
| "她很聪明" | "她扫了一眼代码，'第三行，少了个分号'" |

### 检查清单

每章完成后问：
- [ ] 有没有直接说"XX 很 XX"的句子？
- [ ] 能不能用动作/对话替代？
- [ ] 读者能不能自己感受到，而不是被告知？

---

## 8. 冲突驱动剧情

### 每章必须有冲突或转折

### 冲突类型

| 类型 | 示例 |
|------|------|
| **人与人** | 争吵、对抗、背叛 |
| **人与环境** | 恶劣天气、危险地形、时间压力 |
| **人与自己** | 内心挣扎、道德困境、恐惧 |
| **人与社会** | 规则束缚、偏见、压迫 |

### 章节冲突检查

```
每章至少包含 1 个冲突：

□ 开场冲突（引入问题）
□ 中段冲突（升级矛盾）
□ 结尾冲突（制造悬念）
```

### 示例结构

```
第 X 章结构：
├── 开场：平静日常（20%）
├── 冲突 1：小问题出现（30%）
├── 冲突 2：问题升级（30%）
└── 转折：意外发展（20%）→ 钩子
```

---

## 9. 悬念承上启下

### 每章结尾必须留下钩子

### 钩子类型

| 类型 | 示例 |
|------|------|
| **问题钩子** | "门后是谁？" |
| **行动钩子** | "他举起了枪。" |
| **对话钩子** | "'你根本不知道真相。'她说。" |
| **发现钩子** | "信的最后，署名让他瞳孔一缩。" |
| **时间钩子** | "距离爆炸还有 3 分钟。" |

### ❌ 弱结尾 → ✅ 钩子结尾

```
❌ 弱结尾：
他回到家，躺在床上，想着今天发生的事。
（读者：哦，然后呢？）

✅ 钩子结尾：
他回到家，刚躺下，手机响了。
陌生号码。
"别睡，"对方说，"我们知道你做了什么。"
```

### 章末检查

```
□ 本章结尾有钩子吗？
□ 读者会想翻到下一页吗？
□ 钩子与下一章开头衔接吗？
```

## 📋 深度润色检查清单

### 初稿完成后逐项检查

```
## AI 味检查
□ 删除"璀璨/瑰丽/绚烂"等过度修饰词
□ 替换抽象陈述为具体动作
□ 四字成语不超过 3 个
□ 对话口语化，无书面语套话

## 节奏检查
□ 长句 30% / 中句 50% / 短句 20%
□ 朗读流畅，无喘不过气的长句
□ 句式有变化，不单调

## 细节检查
□ 关键场景有 3-5 个感官细节
□ 视觉/听觉/嗅觉/触觉/味觉运用
□ 细节具体，不笼统

## 展示 vs 讲述
□ 无"XX 很 XX"的直接讲述
□ 情绪用动作/对话展示
□ 读者能自己感受到

## 剧情驱动
□ 本章至少有 1 个冲突
□ 冲突有升级
□ 本章有转折

## 悬念钩子
□ 章末有钩子
□ 钩子吸引读者继续
□ 与下章衔接
```

## 🔧 润色流程

```
1. 生成初稿
   ↓
2. 放置 30 分钟（冷却）
   ↓
3. 第一遍：删除 AI 词（璀璨/瑰丽等）
   ↓
4. 第二遍：抽象→具体（情感→动作）
   ↓
5. 第三遍：对话口语化
   ↓
6. 第四遍：节奏调整（长短句）
   ↓
7. 第五遍：添加感官细节
   ↓
8. 第六遍：检查冲突和钩子
   ↓
9. 朗读检查
   ↓
10. 定稿
```
