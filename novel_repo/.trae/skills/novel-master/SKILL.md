# 长篇小说创作大师

专为创作数百万字长篇小说设计的系统化技能。

## 🎯 核心能力

| 能力           | 说明                                    |
| -------------- | --------------------------------------- |
| **整体规划**   | 三幕式/英雄之旅大纲，分卷设计，章节规划 |
| **角色管理**   | 角色档案、关系图、成长弧线、出场追踪    |
| **世界观构建** | 地理、历史、规则、文化、经济系统        |
| **章节写作**   | 根据大纲生成内容，保持文风统一          |
| **一致性检查** | 剧情矛盾、人物行为、时间线、设定冲突    |
| **伏笔追踪**   | 记录所有线索和回收状态                  |

## ✅ 输出标准与评估

章节必须满足以下硬性指标，否则视为未完成：

- 叙事推进点 ≥ 1（推动事件或局势变化）
- 关系变化点 ≥ 1（人物关系升温/裂开/错位）
- 认知变化点 ≥ 1（角色认知更新或误判）
- 冲突 ≥ 1（外部或内部冲突，非纯情绪叙述）
- 章末钩子 ≥ 1（未解问题或更大危险）
- 章末钩子必须具体、可见、可行动，避免命运宣告式旁白
- 关键事实在本章内前后一致（称谓、亲属关系、物品数量、权力边界）
- 关键信息分层揭示
- 两章相邻的内容不要有重复，后一章应该紧接前一章的推进点产生新的变化
- 重大真相必须以“线索—误判—校正”三段式推进，禁止一章内完成闭环式揭秘
- 长篇结构需维持主线推进与支线分离，避免主线断档超过 5 章
- 生成大纲与章节时必须引用 references 中对应模板与规则
- 主角不得连续 3 章无固定配角互动
- 主角必须拥有固定伙伴，并在前 10 章完成第一次并肩行动
- 感情线为主线之一，必须持续推进且与主线冲突绑定
- 正文不得保留任何模板占位符或空白字段（如 [正文内容] / [待填写]）
- 章节字数目标为 2200-3000 字
- 除非用户明确指定，否则按章节字数目标执行
- 若与润色指南等非硬性清单冲突，以本节与章节输出格式为准

## 🗂️ 工作区优先模式

若用户已经在一个长篇小说工作区内创作，优先遵循工作区文件，而不是泛化想象：

- 必读文件优先级：rules.md → 00-overview.md → 02-world/ → 01-characters/ → 03-volumes/ → 05-tracker/
- 若工作区已存在专用术语表、称谓表、追踪表，必须先读取后再写正文
- 若项目已有章节格式，继续沿用现有格式，不要强行替换为新模板
- 若工作区存在“记忆更新清单”“本章备注”“伏笔追踪”等机制，生成内容时必须同步维护
- 若项目规则与本技能默认值冲突，以项目规则为准

## 🧭 推荐工作流

将长篇创作拆成两个层面：项目搭建与章节生产。

### 项目搭建顺序

1. 明确方向：题材、风格、核心卖点、主线目标
2. 搭建结构：创建总览、角色、世界观、卷纲、章节、追踪文件
3. 生成专属规则：只写本书特有的文风、称谓、术语和禁忌
4. 建立角色档案：补全身份、行为模式、声线、关系网络、转折条件
5. 建立世界观：每条规则写清限制、代价、破坏后果
6. 制定卷纲：卷主题、阶段目标、章节表、高潮、伏笔表

### 章节生产顺序

1. 骨架：先写标题、视角、冲突、剧情节点
2. 正文：只按当前骨架扩写，不擅自改主走向
3. 润色：去模板感、补动作和感官、收紧对话
4. 质检：先报告问题，再决定是否修复
5. 同步：把角色状态、规则变化、阶段目标、伏笔变化写回追踪文件

### 适用场景

- 用户要快速出稿：允许一轮完成“骨架 → 正文 → 润色 → 质检 → 同步”
- 用户要强控结构：拆成多个阶段逐步确认
- 用户只要检查：只输出问题与建议，不改正文

## 📜 项目规则生成规范

为单个项目生成 rules.md 时，优先只写“本书特有、可执行、能约束正文”的规则，不重复通用写作常识。

rules.md 至少覆盖以下模块：

1. 文风与节奏
2. 对话与心理描写纪律
3. AI 味检查清单
4. 称谓与用语统一
5. 错别字、语法与逻辑检查
6. 章节固定分区结构

### 1. 文风与节奏

- 明确文风基调定位，如冷峻、沉郁、明快、压迫、诗性等
- 明确段落长度、长短句比例、对话占比、正文字数目标
- 明确章末收束方式和禁止使用的结尾套路

### 2. 对话与心理描写纪律

- 为核心角色写出具体声线约束：语气基调、句长倾向、口头禅、禁用表达
- 明确抽象情感如何改写为动作、停顿、呼吸、体感、操作失误或环境折射
- 明确解释性台词如何拆进审讯、交易、命令、争执等场景

### 3. AI 味检查清单

- 分成“禁用词与限频词 / 句式与结构 / 密度与节奏 / 对话真实性”四组
- 每组优先给出可检查阈值，而不是泛泛描述
- 若用户要求去 AI 味，优先处理总结句、连续比喻、抽象情感直述、命运宣告式结尾、空洞对话标签

### 4. 称谓与用语统一

- 可使用三层称谓：叙事层 / 对话层 / 元数据层
- 为项目专属术语建立最小术语表，防止章节里出现同义混写
- 若称谓规则与角色档案冲突，先指出冲突源，再决定保留哪一处

### 5. 错别字、语法与逻辑检查

- 明确项目中的高风险同音字、常见误写术语、因果链漏洞与时间空间锚点要求
- 规则一旦落到正文，必须能被验证、阻拦、追责或反噬，不能只当背景介绍

### 6. 章节固定分区结构

- 新项目优先推荐以下结构：

```md
## 正文

## 记忆更新清单

## 本章备注
```

- 旧项目若已形成稳定历史结构，可写兼容说明，但不要在同一项目中随意混用两套新旧模板

## 👤 角色档案规范

角色档案应优先维护为“可执行档案”，不是人物简介。每个角色至少包含以下 7 组信息：

### 1. 基本信息

- 姓名
- 别名 / 乳名 / 代号
- 性别
- 年龄
- 外貌特征（只留辨识度最高的 2-3 个）
- 身份 / 职业
- 阵营
- 社会位置（在权力结构中的层级，手上筹码）

### 2. 性格与行为

- 核心性格（2-3 个关键词）
- 行为模式（压力 / 背叛 / 被示好时的条件反射）
- 动机（当前最强驱动力）
- 价值观底线（绝不会做什么）
- 盲区 / 认知偏差（误判如何驱动剧情）
- 情感弱点（什么人或事能让他失去理性）

### 3. 声线与说话风格

- 语气基调
- 口头禅 / 高频词
- 句长倾向
- 称谓习惯
- 不会说的话（反面约束）

### 4. 能力与装备

- 当前境界 / 等级
- 核心能力
- 武器 / 关键道具
- 弱点 / 限制（条件、风险、消耗）
- 战斗风格
- 成长潜力（隐藏能力、觉醒条件）

### 5. 关系网络

- 与主角关系
- 与其他角色关系（性质 + 信任度 + 利益绑定或冲突）
- 情感锚点
- 欠债关系
- 秘密（隐瞒了什么，暴露后会发生什么）

### 6. 角色弧线与剧情定位

- 叙事功能
- 出场状态
- 核心矛盾
- 转折条件
- 预期终局方向
- 关键事件经历
- 伏笔 / 悬念

### 7. 当前状态

- 出场章节
- 当前状态（活 / 死 / 失踪 / 变化中）
- 已完成的变化
- 待解决的问题

### 角色档案执行规则

- 角色能力必须符合世界观上限，不得超纲
- 关系网络必须双向一致，修改 A 对 B 的关系时同步检查 B 对 A
- 新角色创建后，应同步登记到总览中的角色清单
- 若章节导致角色状态、关系或阶段位置变化，应回写档案

## 🧱 章节输出格式

每章必须按以下顺序输出：

1. **开场锚点**：时间/地点/视角/目标
2. **主要场景**：动作链（动作 → 反应 → 后果）
3. **对话段**：至少 1 处，符合角色声线
4. **推进点**：明确的剧情推进句
5. **章末钩子**：未解决或即将升级的事件

要求：

- 开场锚点必须以一句话显式写出，不可隐含
- 章节末尾必须输出“记忆更新清单”
- 开场锚点必须紧接正文第一行出现，格式：开场锚点：时间/地点/视角/目标
- 正文必须包含明确的“推进点”句，使用“推进点：...”标记
- 章末钩子必须单独成段，使用“钩子：...”标记，并作为正文最后一段

### 结构兼容规则

- 若项目已采用标题分区结构，可使用“## 正文 / ## 记忆更新清单 / ## 本章备注”
- 若项目已采用行内标记结构，可继续使用“开场锚点 / 认知变化点 / 关系变化点 / 推进点 / 钩子”
- 不要在同一项目中随意混用两套结构；优先保持历史章节一致

## 📚 分卷大纲规范

分卷大纲优先采用可检索、可核对、可同步的结构。每卷至少包含以下 7 组 18 项内容：

### 一、卷元信息

1. 卷标题与主题
2. 时间跨度
3. 主要地点
4. 章数规划（章节号必须与全书连续）

### 二、目标与冲突

5. 核心冲突
6. 卷内阶段目标（开端 / 发展 / 高潮三个递进目标）
7. 主线定位
8. 卷末主问题

### 三、角色规划

9. 关系变化目标
10. 认知变化目标
11. 本卷新增角色表（角色 / 性别 / 外貌 / 身份 / 首登章节）
12. 本卷固定配角与关系类型表（角色 / 关系类型 / 卷内功能）
13. 角色弧线节拍（关键角色跨章情绪 / 行为轨迹，落到具体章节）

### 四、章节列表

14. 章节表（章节 / 标题 / 状态 / 字数 / 概要）

### 五、事件与高潮

15. 本卷重要事件（编号并标注章节）
16. 卷高潮设计（高潮事件 + 代价 + 未解问题）

### 六、世界观落地

17. 卷内规则定义（本卷涉及的规则参数、等级上限、资源消耗、封控级别等）

### 七、伏笔布局

18. 本卷伏笔表（编号 / 内容 / 类型 / 触发方式 / 显隐 / 章节 / 状态 / 回收章节）

### 卷纲执行规则

- 每卷必须有卷主题、阶段目标、章节列表、高潮设计、卷末钩子
- 每 10 章至少 1 次小高潮
- 跨卷伏笔须在下一卷前 15 章内至少呼应一次
- 感情线每 5 章至少推进或转折一次
- 主角不得连续 3 章无固定配角互动
- 更新卷纲后应同步维护卷目录索引

## 🧭 视角与声线纪律

- 单章固定视角，不允许无标记切换
- 每个角色拥有固定语气、口头禅和句长倾向
- 对话必须符合角色价值观，不允许中性万金油语气

## 🧮 一致性检查细则

检查必须输出三列：问题点 / 证据段落 / 修正建议

必查项：

- 动机与行为冲突
- 能力成长速度不合理
- 时间跨度与地理距离不匹配
- 社会规则与经济逻辑自相矛盾
- 设定被忽略或被改写
- 人称称谓与亲属关系一致
- 物品稀缺性与行为动机一致
- 权力边界与角色反应一致
- 伤势与行动能力一致
- 关键信息揭示节奏一致
- 境界差距与胜负结果一致
- 重复段落与重复句式已清理
- 主线阶段目标连续
- 支线引入后 10 章内必须推进或收束
- 角色动机在 20 章跨度内保持一致
- 配角出场频率与主线推进相关
- 配角行为动机与利益关系一致
- 感情线推进与主线事件因果一致
- 情感变化有明确触发事件与代价
- 关键角色行为必须与角色档案一致，若违背必须给出可追溯的动机变化与代价
- 角色能力提升或战斗强度跃迁需有明确铺垫与代价，不得跨章直接拔高

## 🔍 专项质检清单

除通用一致性检查外，支持以下专项审查：

- 规则文件审查：检查 rules.md 是否完整、可执行、与世界观和总览一致
- 角色档案审查：检查信息完整性、关系双向一致、能力与设定匹配
- 世界观审查：检查规则完整性、术语统一、代价是否落地
- 卷纲审查：检查章节编号、高潮节拍、伏笔布局、跨卷衔接
- 跨域一致性审查：世界观 × 角色 × 卷纲 × 章节 × 伏笔的交叉核验
- 风格漂移审查：抽样多章比对句长、对话占比、角色声线、结尾方式

输出时优先给问题、证据、严重度与修正建议，不要先写泛泛总结。

## 📈 揭示节奏与战力规则

- 前 10 章只揭示主线的 1-2 层信息，不揭示终局级身份与最终真相
- 前 20 章禁止揭示终局级真相的核心结构与完整运行机制，只能出现碎片线索
- 重大身份揭示后，至少隔 3 章再揭示下一层重大真相
- 前 10 章不出现超越主角 2 个大境界的正面胜利
- 前 10 章的越阶胜利最多 1 次，且必须付出明显代价
- 关键设定必须逐步验证，不允许一次性全部说明
- 重大真相揭示必须与角色的付出与失去绑定，禁止纯信息灌输
- 每 10 章出现 1 次小高潮
- 每 50 章出现 1 次卷内大高潮
- 每卷结尾必须完成 1 次主线阶段性闭环并引出下一卷主问题

## 🧷 事实锁定清单

- 主角身世关系是否已定稿
- 导师身份是否已定稿
- 关键反派立场是否已定稿
- 关键功法来源是否已定稿
- 境界层级与战力差距是否已定稿
- 主线目标是否已定稿
- 卷内阶段目标是否已定稿

## 🧠 记忆文件加载规则

- 生成大纲与章节前必须读取项目记忆文件，缺失则要求用户补充
- 必读文件：rules.md、00-overview.md、01-characters/、02-world/、03-volumes/、05-tracker/foreshadowing.md
- 读取后必须输出摘要：主线目标、当前卷阶段目标、关键角色动机、世界规则、已登记伏笔
- 若存在 05-tracker/quotes.md，可作为表达层风格样本与金句回收表读取

## 🧑‍🤝‍🧑 协作与节奏闸门

- 多人协作生成章节必须在“记忆更新清单”后追加“节奏闸门自检”
- 自检字段必须包含：本章重大真相数量 / 真相阶段（线索/误判/校正）/ 距离上一条重大真相章节数 / 是否触及终局级结构 / 角色付出与失去 / 可验证线索
- 任一字段不满足“揭示节奏与战力规则”视为未完成，必须重写
- 单章只能推进真相链条的一个阶段，禁止同章完成闭环
- 协作产出必须引用前文章节编号，未标注视为未完成
- 新增设定与能力跃迁必须在“记忆更新清单”中登记来源与代价

## 📝 记忆更新规则

- 每章生成后必须输出“记忆更新清单”，包含新增/变更条目
- 更新清单必须覆盖：角色状态、世界规则变更、阶段目标推进、伏笔新增或回收
- 记忆更新清单需写回 00-overview.md 与 05-tracker/foreshadowing.md
- 若本章使角色状态、关系、出场阶段或未解决问题发生变化，应同步更新对应角色档案
- 若无法直接写回文件，必须输出可直接粘贴的更新块，分别标注写入位置

## 🏷️ 称谓与术语治理

- 支持三层称谓治理：叙事层、对话层、元数据层可以不同，但必须全书一致
- 角色改名或术语替换前，先扫描全书影响范围，再输出影响报告，再执行替换
- 批量替换后必须做残留检查、误伤检查和变体检查
- 若旧名是其他词的子串，必须先警告再替换
- 世界观术语优先维护专有名词表，避免章节里出现同义混写

## 大纲生成规范

- 总体大纲必须包含：核心矛盾、主线目标、主线阶段目标、卷数规划
- 分卷大纲必须包含：卷主题、卷内阶段目标、卷内高潮、卷末主问题
- 章节大纲必须包含：目标、阻碍、行动、反转、代价、章末钩子
- 每 10 章输出一次“剧情推进点/关系变化点/认知变化点/冲突升级方式”汇总
- 汇总仅做结构性统计，不新增设定与真相
- 大纲输出必须按 references/chapter-outline-template.md 的字段顺序组织

## 🧠 宏观结构与主题回环

- 全书主线需拆成 3-5 个阶段目标，每阶段完成一次问题闭环
- 每卷必须围绕 1 个核心命题展开，并在卷末形成正反变奏
- 关键主题词必须在每 30 章至少出现 1 次变体回应
- 世界规则必须在 3 个不同层级以不同代价被验证

## 🎭 角色长期动机与转折条件

- 主要角色必须维护“动机-代价-选择”三段链条
- 角色转折必须具备至少 2 个前置条件并在 10 章内铺垫
- 重要角色每卷至少 1 次价值观冲突与再选择
- 角色成长必须体现“能力提升-代价升级-关系重排”同步变化

## 🧬 跨卷伏笔矩阵

- 关键伏笔必须标注“首次出现/二次呼应/回收章节/回收方式”
- 跨卷伏笔必须在下一卷前 15 章内完成一次呼应
- 关键反派或终局设定必须通过 3 次递进式线索推进

## 🪝 伏笔管理规则

- 每章新增伏笔必须登记编号与回收计划
- 伏笔首次出现后，最多 30 章内必须回收或升级为主线
- 伏笔回收必须与当前冲突或角色选择直接相关

## 🎛️ 风格控制补充

- 删除解释性对话，设定信息优先拆进交易、审讯、命令、驳回、清算等场景里
- 减少空洞对话标签，优先使用动作、停顿、手势、呼吸承接语气
- 避免高频意象词反复出现，尤其是同一章内对同一色彩、气味、光影的重复调用
- 连续三章不要使用相同类型的章末钩子
- 若用户要求“去 AI 味”，优先处理：总结式段尾、抽象情感直述、对称排比、连续比喻、命运宣告式结尾

## 🌍 世界观设定规范

世界观应优先写成“可验证规则系统”，不是散文式百科。完整版本建议覆盖以下 7 大类 29 项：

### A. 核心体系与规则

1. 核心命题
2. 世界规则 / 法则（每条规则必须包含限制 + 代价 + 破坏后果）
3. 能力与资源体系（稀缺资源、获取方式、使用代价）
4. 等级 / 境界体系（上限、瓶颈、升级路径与代价）
5. 规则约束与禁忌
6. 规则的元层（规则是谁定的、能否修改、修改的代价）

### B. 地理与环境

7. 地形地貌
8. 环境特征
9. 空间结构
10. 空间关系

### C. 社会体系

11. 权力结构（至少 3 层，并标注制衡关系）
12. 经济体系
13. 法律与秩序
14. 阶层流动

### D. 文化与认知

15. 信仰与宗教
16. 习俗与仪式
17. 称谓与语言
18. 知识传承

### E. 历史与时间

19. 纪年与大事件
20. 历史断层
21. 遗迹与传承
22. 时间体系

### F. 生命与死亡

23. 种族与生命形态
24. 死亡与存续
25. 生活细节

### G. 悬念与伏笔

26. 隐藏秘密
27. 线索体系
28. 潜在冲突
29. 可验证场景（至少 3 个）

### 世界观执行规则

- 新增规则不得与已有规则冲突；若冲突，先明确指出并请求确认
- 核心术语必须与项目规则、角色档案和总览一致
- 修改权力结构时，应检查相关角色档案是否需要同步更新
- 当单文件世界观过大时，可拆成子文件，但主索引必须保留

## 🧭 阶段目标生成规范

- 全书阶段目标必须输出 3-5 个阶段
- 每阶段目标必须包含：目标、阻碍、代价、阶段性闭环、下一阶段主问题
- 阶段目标必须与世界规则发生正面冲突
- 每阶段至少 1 次不可逆损失
- 每卷至少 1 次价值观冲突与再选择
- 每阶段至少 1 个配角推动的主线转折
- 每阶段至少 1 次感情线关键推进或重大受挫

## 🧑‍🤝‍🧑 配角网络与群像规则

- 每卷必须设定 4-6 个固定配角，并标注与主角的关系类型
- 每 5 章至少 1 次由配角发起的关键行动或信息输入
- 配角必须具备独立目标，且与主线存在利益冲突或协同
- 至少 1 条配角支线在 20 章内完成闭环

## 💞 感情线规则

- 感情线必须设定“起点—升温—裂痕—选择—代价—阶段性结果”
- 每 5 章至少 1 次情感互动或关系转折
- 禁止用误会或降智强行推进
- 感情线必须影响主角关键选择或主线走向

## 🎣 伏笔回收规则

伏笔必须包含回收计划：

- 伏笔点 / 首次出现章节 / 预期回收章节 / 当前状态
- 未设定回收章节的伏笔视为无效

## 🔥 深度润色指南

**重要**: 生成章节后必须进行深度润色，去除 AI 味。

### AI 味检查清单

- [ ] 删除"璀璨/瑰丽/绚烂"等词
- [ ] 抽象情感 → 具体动作
- [ ] 四字成语 ≤3 个
- [ ] 对话口语化
- [ ] 长短句 3:5:2 比例
- [ ] 五感细节 3-5 个
- [ ] 展示而非讲述
- [ ] 符合“输出标准与评估”与“章节输出格式”的硬性要求
- [ ] 信息揭示不过载（单章最多 1 个重大真相）
- [ ] 每 10 章有阶段性回收与推进

### AI 味负面清单

- [ ] 过度总结式旁白
- [ ] 情绪形容词堆叠
- [ ] 世界观硬讲解
- [ ] 缺少行动后果
- [ ] 万能中性语气
- [ ] 命运宣告式结尾与固定句式（如“他不知道的是……从这一刻起，他的人生将彻底改变”）

### 复写要求

- [ ] 删除解释性句子，改为场景动作
- [ ] 将情感词替换为行为或环境细节
- [ ] 将“告诉读者”改为“让读者看到”

完整指南：见 `references/deep-polish-guide.md`

## ⚠️ 结尾句式多样性（重要更新）

### 禁止使用的结尾句式

```
❌ "他不知道的是，从这一刻起……"
❌ "而此时的他还不知道……"
❌ "他更不知道的是……"
❌ "从此，他的人生改变了……"
❌ "命运的齿轮开始转动……"
```

### 正确的结尾方式（轮换使用）

| 类型         | 示例                           | 适用章节 |
| ------------ | ------------------------------ | -------- |
| **动作钩子** | "他推开门，愣住了。"           | 动作章   |
| **对话钩子** | "'你根本不知道真相。'她说。"   | 对话章   |
| **问题钩子** | "门后是谁？"                   | 悬念章   |
| **场景钩子** | "远处，火光冲天。"             | 过渡章   |
| **发现钩子** | "信的最后，署名让他瞳孔一缩。" | 揭秘章   |

### 检查规则

- 连续三章不能用相同结尾方式
- 禁止上帝视角剧透
