# Compatibility Notes

novel-master 当前已经具备较完整的 skill 主规范，但仓库内不同层级的材料成熟度并不完全一致。

## 当前状态

### 已对齐较好的部分

- SKILL.md：长篇创作协议、工作区优先模式、角色/世界/卷纲/章节规范较完整
- README 与贡献文档：已补齐开源使用所需的入口说明

### 仍偏轻量脚手架的部分

- scripts/novel-cli.js：当前主要负责创建目录、生成占位模板、输出检查框架
- assets/templates/：当前模板更接近早期脚手架版本
- references/：提供的是通用参考模板，不完全等同于最新版 skill 协议

## 这意味着什么

- 如果你把 novel-master 当成“创作规则 skill”使用，当前主规范已经足够强
- 如果你把 novel-master 当成“开箱即用的完整本地创作工具链”，CLI 与模板资源仍需要继续升级

## 已知差异示例

- SKILL.md 当前默认章节字数口径为 2200-3000 字
- 部分早期参考模板仍保留更宽松或更旧的字段设计
- 主规范强调工作区优先和同步回写，但 CLI 目前只做文件脚手架，不做真正的同步流程

## 开源发布建议

当前项目已作为开源项目发布。项目定位为：

- 一个以 skill 规则为核心的长篇创作框架
- 附带轻量 CLI 和基础模板
- CLI 与模板资源会在后续版本中逐步追平主规范
